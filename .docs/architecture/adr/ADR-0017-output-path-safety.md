# ADR-0017: Unified Output Inventory and Realpath-Safe Writes

## Status

Accepted

## Context

AgentQuilt can produce document targets, provider-adapter files, and the lock
file in one build. Validating those output families independently permits two
unsafe cases: different producers can claim the same portable path, and a
lexically contained output can traverse an existing symlink to write outside
the repository. A force flag must not bypass either invariant because force
only expresses permission to replace content at an otherwise valid path.

## Decision

Before reading or writing any generated output, `build` and `check` create one
inventory containing document outputs, every adapter output, and the lock
path. A shared validator then enforces all of these rules:

1. Every configured output is relative and resolves lexically inside the
   repository root under native, POSIX, and Windows path semantics.
2. Portable path keys normalize path separators, Unicode normalization, and
   case. Two producers cannot claim the same portable path.
3. No path component below the repository root may be a symbolic link. This
   includes dangling links and links whose targets remain inside the
   repository. Existing paths and the nearest existing parent are also
   real-path checked before directories are created.
4. Validation completes for the full inventory before any output or lock file
   is read, created, or changed.
5. `--force` does not bypass path containment or collision checks.

The same validator is used by `build` and `check`, so CI evaluates the same
path rules as local generation.

## Consequences

Positive:

- Document and provider targets cannot silently overwrite one another.
- A symlinked output path cannot redirect generated bytes outside the
  repository, onto canonical source, or onto another generated destination.
- Rejected builds are atomic with respect to generated outputs and the lock.
- Path behavior is deterministic across common case and Unicode differences.

Negative:

- Projects relying on any symlink component in a generated output path are
  rejected with no escape hatch.
- Case-distinct output paths that work on a case-sensitive filesystem are
  rejected to preserve portable checkout behavior.
- Backslash traversal, drive-letter paths, and UNC paths are rejected even on
  POSIX hosts so a configuration cannot change meaning across operating
  systems.
