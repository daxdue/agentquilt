# AgentQuilt Project Plan

## Overview
AgentQuilt is a framework for maintaining AI agent instruction files in distributed engineering teams. This document outlines the complete work breakdown from the specification, organized by the 4-week rollout plan.

**Current Status:** Pre-release, placeholder version  
**Target Completion:** 4 weeks from kickoff  
**Repository:** https://github.com/daxdue/agentquilt

---

## 4-Week Rollout Timeline

### Week 1: Prototype (Foundations)
Build core compiler and schema validation.

#### 1.1 Schema Definitions (TypeScript/Zod)
- [ ] Define `AgentManifest` schema
  - Fields: id, name, version (semver), status, maturity, owners, description, tags, sections, contexts, compile, imports
  - Validation: semver format, min owners=1, sections as objects with id/title/order
- [ ] Define `InstructionBlock` schema
  - Fields: id, agent, type, section, owner, status, priority, risk, summary, rationale, text, tags, applies_when, conflicts_with, supersedes, created_by, updated_by, traceability
  - Risk defaults to "medium" at load (not in YAML)
  - Structured applies_when: `{ context, channel? }`
- [ ] Define `Eval` schemas
  - PromptPresenceEval: id, agent, name, expected (must_include/must_not_include)
  - LiveLlmEval: id, agent, name, input, expected (rubric, minimum_score)
- [ ] Define enums: Status, Type, Risk, Channel, Maturity
- [ ] Add SemVer validation: `/^\d+\.\d+\.\d+$/`
- **Owner:** Core team  
- **Status:** Not started

#### 1.2 Core Compiler Implementation
- [ ] Implement `loadAgent()` - load and validate manifest.yaml
- [ ] Implement `loadBlocks()` - load all instruction blocks from instructions/ directory
- [ ] Implement deterministic block sorting
  - Sort by: section order → priority → block id (Unicode code point)
  - Critical: Do NOT use `localeCompare()`
- [ ] Implement Markdown renderer
  - One section per heading (uses section.title)
  - Blocks under section in sorted order
  - Normalize blank lines, trim trailing spaces
  - Preserve code block indentation
- [ ] Implement source hash computation
  - SHA-256 of canonical manifest + sorted blocks
  - Normalize line endings (LF) for cross-platform consistency
- [ ] Implement generated file header
  ```
  <!--
  Generated by agentctl. Do not edit this file manually.
  Source: agents/{agent-id}/
  source-hash: sha256:<hash>
  agentctl-version: <version>
  -->
  ```
- **Owner:** Core team  
- **Status:** Not started  
- **Dependencies:** 1.1 complete

#### 1.3 Agentctl CLI - Compile Command
- [ ] Implement `agentctl compile <agent-path>`
  - Load manifest + blocks
  - Filter by status (deprecated never, draft with --include-draft)
  - Sort deterministically
  - Render Markdown
  - Compute source hash
  - Write to `agents/{id}/generated/{id}.md`
- [ ] CLI output: "Compiled agent {id} | Blocks: {count} | Hash: {hash}"
- [ ] Add `--include-draft` flag
- [ ] Add `--include-block-comments` debug flag
- **Owner:** Core team  
- **Status:** Not started  
- **Dependencies:** 1.2 complete

#### 1.4 Sample Agent Migration

### Week 2: Validation and CI (Enforcement)
Add linting, verification, and CI/CD integration.

#### 2.1 Agentctl Lint Command
- [ ] Implement `agentctl lint <agent-path>`
- [ ] Core lint rules (ERRORS):
  - `manifest.required-fields`: All required manifest fields present
  - `manifest.valid-sections`: Sections are objects with id/title/order
  - `manifest.valid-version`: Version matches semver
  - `block.required-fields`: All required block fields present
  - `block.unique-id`: Block IDs globally unique across repo
  - `block.valid-agent`: Block.agent matches manifest.id
  - `block.valid-section`: Block.section in manifest.sections
  - `block.valid-status`: Status is valid enum value
  - `block.valid-type`: Type is valid enum value
  - `block.valid-priority`: Priority is numeric
  - `block.deprecated-has-supersedes`: Deprecated blocks must have supersedes
  - `block.supersedes-target-exists`: All supersedes IDs resolve
  - `block.conflict-reference-valid`: All conflicts_with IDs resolve
  - `block.rationale-required-for-high-risk`: High/critical blocks need rationale
- [ ] Optional lint rules (WARNINGS):
  - `block.max-size`: Warn if block > 250 words
  - `block.no-global-always-without-context`: Flag broad global rules
  - `block.no-duplicate-summary`: Flag duplicate summaries
  - `block.legacy-section-present`: Flag legacy section usage
  - `block.owner-required-from-codeowners`: Validate against CODEOWNERS
- [ ] Output format:
  ```
  Agent: {id}
  Status: [passed|failed]
  
  ERROR {block-id}: {message}
  WARN {block-id}: {message}
  ```
- **Owner:** Core team  
- **Status:** Not started  
- **Dependencies:** 1.1 complete

#### 2.2 Agentctl Verify Command
- [ ] Implement `agentctl verify <agent-path>`
- [ ] Recompute source hash from current source files
- [ ] Recompute Markdown body from current source
- [ ] Compare against existing generated file header
- [ ] Fail if hash mismatch or body mismatch
- [ ] Exit 0 on success, 1 on mismatch
- [ ] Output: Source hash, verification status
- **Owner:** Core team  
- **Status:** Not started  
- **Dependencies:** 1.2 complete

#### 2.3 Lint Rule: Repository-Wide Duplicate ID Detection
- [ ] Scan all `agents/*/instructions/*.yaml` for block IDs
- [ ] Scan imports for shared block IDs
- [ ] Error if any ID appears more than once
- [ ] Include shared block namespace: `shared.*`
- [ ] Report conflicting IDs with locations
- **Owner:** Core team  
- **Status:** Not started  
- **Dependencies:** 2.1 complete

#### 2.4 GitHub Actions Workflow
- [ ] Create `.github/workflows/validate-agents.yml`
- [ ] Triggers: `on: [pull_request] paths: ["agents/**", "tools/agentctl/**"]`
- [ ] Steps:
  1. Checkout
  2. Setup Node 22
  3. npm ci in tools/agentctl
  4. `agentctl lint ../../agents` - fail on error
  5. `agentctl verify ../../agents` - fail if inconsistent
  6. `agentctl test ../../agents` (once evals ready)
- [ ] CI fails if:
  - Manifest invalid
  - Block invalid
  - Duplicate IDs
  - Verify detects inconsistency
  - Required evals fail
  - High/critical blocks missing rationale
- **Owner:** DevOps/Core team  
- **Status:** Not started  
- **Dependencies:** 2.1, 2.2 complete

## MVP Scope Summary

### ✓ Included in MVP
- Agent manifest format (sections, semver version, compile config)
- Instruction block format (risk defaulting, applies_when, references)
- Deterministic Markdown compiler (locale-independent)
- Basic linter (required fields, schema validation, references)
- Source/generated integrity via verify + content hash
- Basic semantic diff output
- Prompt_presence eval format and runner
- GitHub Actions CI integration
- Sample and real agent migration
- PR template and CODEOWNERS
- Documentation and guides

### ✗ NOT in MVP
- Web UI
- Database
- Full LLM-based grading (live_llm)
- Automatic semantic conflict resolution
- Multi-repository synchronization
- Agent package publishing
- Telemetry/analytics

## Key Decisions

1. **Generated files are not committed** - they are build outputs, produced at deploy time
2. **Block IDs are globally unique** - across entire repo, not per-agent
3. **Risk defaults to medium** - at load time, ensures explicit review tiers
4. **Deterministic compilation** - critical for cross-platform CI consistency
5. **No locale-specific sorting** - use Unicode code point comparison only
6. **Structured applies_when** - enables deterministic conflict checking

## Success Criteria (Acceptance)

1. ✓ Developers can add/modify instructions without editing final .md manually
2. ✓ agentctl compile generates deterministic .md files
3. ✓ agentctl lint detects invalid manifests and blocks
4. ✓ CI fails (via agentctl verify) when compiled output is inconsistent
5. ✓ CI fails when instruction IDs are duplicated
6. ✓ Pull requests show semantic change summaries
7. ✓ At least one real agent migrated successfully
8. ✓ 3+ evals exist for migrated agent
9. ✓ Generated prompt output is reproducible
10. ✓ Team understands instruction blocks are source of truth
11. ✓ No duplicate IDs repository-wide
12. ✓ All supersedes/conflicts_with references resolve
